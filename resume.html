My research interests include operating systems and programming languages with a particular emphasis on heterogeneous high-performance computing. To begin with, I have developed a runtime and driver module for GPUs, one of popular heterogeneous compute devices for massively parallel computing. Particularly I am involved in a project on GPU virtualization leveraging Linux and Xen. Though GPUs are currently black box devices due to vendor's proprietary reasons, I have made everything open about how to virtualize GPUs in the hypervisor, using Linux open-source GPU runtime and driver modules. I plan to extend my project toward a more generalized class of heterogeneous high-performance computing with on-board multi-core CPUs (e.g., Intel Xeon Phi) and FPGAs. I believe this is a great contribution to a grander vision of computer science.

Building ECMAScript engine from scratch

https://github.com/Constellation/iv
I built the new ECMAScript engine that conforms ECMA262 spec completely. And I found and reported many bugs in test262, conformance suite from ECMA TC39. It has Register VM and Method JIT compiler for x86_64 environment. Now investigating LLVM MCJIT infrastructure to construst optimizing JIT compiler (tier 2) on the top of method JIT (tier 1).

Building ECMAScript tools infrastructure

https://github.com/Constellation/escodegen

I have been maintaining Esprima ECMAScript parser as a commiter. And I also built escodegen, ECMAScript unparser. They are widely used as an infrastructure of ECMAScript tools such as Google-caja, Mozilla LLJS, Mozilla Shumway etc.

Building JSDoc parser parsing type annotation precisely

https://github.com/Constellation/doctrine

I built doctrine JSDoc parser. It can parse JSDoc type annotation and produce a structured tree. This library is used in Eclipse Orion for its completion. And it's also used for JSDoc checking in ESLint, a ECMAScript lint tool.
